### Database Schema

```prisma
model Launch {
  id         String   @id   // Launch Library 2 UUID
  name       String
  net        DateTime        // NET (No Earlier Than) launch time
  rocketName String
  status     String
  padName    String?
  padLat     Float?
  padLon     Float?
  createdAt  DateTime @default(now())
  updatedAt  DateTime @updatedAt
  satellites Satellite[]
}

model Satellite {
  id        Int      @id @default(autoincrement())
  noradId   Int      @unique
  name      String
  tle1      String
  tle2      String
  updatedAt DateTime @updatedAt
  launchId  String?
  launch    Launch?  @relation(fields: [launchId], references: [id])
}
```

### Key Implementation Notes

#### CesiumJS + Turbopack
- Next.js 16 defaults to Turbopack вЂ” `copy-webpack-plugin` doesn't work.
- Solution: `postinstall` npm script (`scripts/copy-cesium.js`) copies CesiumJS static assets to `public/cesium/` after every `npm install`.
- `CESIUM_BASE_URL` is set to `/cesium/` inside the Globe component's `useEffect`, before the dynamic `import("cesium")`.
- `public/cesium/` is gitignored (regenerated by postinstall).

#### CesiumJS + Next.js App Router
- Globe is loaded with `dynamic(() => import(...), { ssr: false })`.
- In Next.js 16 App Router, `ssr: false` dynamic imports must be in a `"use client"` component вЂ” `page.tsx` has the directive.
- Scene properties (`skyBox`, `sun`, `moon`, `fog`) typed as possibly undefined in CesiumJS вЂ” use null checks before accessing.

#### Prisma 7 Breaking Changes
- `new PrismaClient()` with no args is invalid in Prisma 7 вЂ” requires a driver adapter.
- Use `PrismaLibSql({ url })` from `@prisma/adapter-libsql` (takes a config object, not a pre-created client).
- Generated client entry point is `client.ts`, not `index.ts` вЂ” import from `@/app/generated/prisma/client`.
- `prisma.config.ts` manages the datasource URL (reads `process.env.DATABASE_URL`).
- After deleting/reinstalling `node_modules`, must run `npx prisma generate` to recreate the generated client.

```bash

npx prisma generate  # regenerate Prisma client if node_modules was wiped
npm run dev          # starts at http://localhost:3000
```
